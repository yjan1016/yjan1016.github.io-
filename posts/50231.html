建站目的

我用过很多笔记软件、写作软件、博客写作平台，他们有的免费，有的付费，有的可选择性付费，有的界面很好，有的功能很全，但是存在优点的同时也有缺点。最近常用的那几个有wiz note、notion、flow us、语雀、博客园、CSDN、Gridea。

为知笔记大一、大二用的很多，写了一些笔记，一开始很好，它的界面很漂亮，不用担心存储问题，评论也很好，颜色偏淡，可以更改几种字体，可加密，可导出，分享（分享是付费功能，60r/year）文档的界面也特别好看，分享的文档还可以看到阅读次数，我在热铁盒建站的时候想过将笔记做在为知笔记上，通过超链接移植到网站上，后来改版后我不喜欢，因为图片复制过去经常复制不成功，文档一多就很乱，旧版可以写html文档，但是没有博客园那么好，很多效果无法显示。

notion、flow us不付费界面难看，一个颜色太丑，还有都太臃肿，免费的限制流量，体验效果不好。

CSDN老嫌我文章质量低，虽然但是看了还是会很不爽，他的自由度不高，编辑页面也不好用（简书同样），但CSDN可以引图片过去网站上，语雀和博客园不能。

博客园现在的界面还行，就是文章那块的样子我不喜欢，太大了，分块不好看，搜索功能和分类功能不好，有的时候响应比较慢，虽然自由度比其他任何软件都高，但是他的编辑页面也不好用。CSDN能穿资源，博客园有限制，而且它广告我好像还不知道怎么去，虽然影响不大。

白嫖语雀一年多，但是语雀也有很多不足，自由度不高，后来出现编辑错误，所以就还是打算自己建一个网站用于记录。

用百度网盘同步着，有1G的容量，防止意外发生。离线可正常访问，只是没看板娘和音乐播放器，有些也无法访问，但对于笔记访问不影响。

建站过程

01 hexo建站

  • 安装git和node.js
  • node.js安装
  • hexo文档：hexo文档
  1. 安装hexo：npm install -g hexo-cli
  2. 初始化hexo： hexo init myblog (不一定是myblog，名字随便取都可以。)
  3. 进入文件夹，安装npm：cd myblog，then npm install
  4. 生成静态网页：hexo g
  5. 启动服务器：hexo s

默认是landscape主题

2023-01-17 为了看手机上的效果，hexo+github

hexo s # 通过启动本地服务器，预览文章效果
hexo d # 部署到github中，更新网页端的内容

https://yjan1016.github.io/

百度网盘同步的时候不要用hexo d。用hexo d过几分钟刷新自己的博客网址内容会更新了。

02 front-matter

20230116发现问题：matery主题发布时间变更新日期，文档front-matter 中加入 date没用

现象出现原因：因为我修改了文件名，所以导致电脑中去查看文件的属性就会发现创建时间安和修改时间都是那个时间。

解决方法：思路是修改文档创建时间，可以用ac软件，就是Attribute Changer，这是一个windows专门修改时间的软件，且免费，安装英文版，然后找到你要改的文件，右键点Attribute Changer（不是去点安装后的那个acmain.exe），后面的步骤就不多说了。

front-matter写法注意事项：

  • 写法即在普通md文章中添加--- 这之间的就是front-matter
  • front-matter中的yml中标点符号全部是英文，并且标点符号后需要有空格

Front-matter 选项详解

——来源于主题官方介绍文档

Front-matter 选项中的所有内容均为非必填的。但我仍然建议至少填写 title 和 date 的值。

配置选项 描述 默认值
title 文章标题 文章标题
date 创建日期，最好保证全局唯一 发布时间
author 文章作者 _config.yml 中的 author
img 文章特征图 feature_Images中的某个值
top 文章是否置顶 true
cover 文章是否需要加入首页轮播封面中 false
coverImg 首页轮播时的图 无
password 密码 无(用 SHA256 加密后的密码，在 config.yml 中激活verifyPassword 选项)
toc 目录 true(如果config.yml 中激活了 toc 选项)
mathjax 数学公式支持 false(_config.yml 文件中也需要开启才行)
summary 文章摘要 自动截取文章的部分内容作为摘要
categories 文章分类 宏观上大的分类，只建议一篇文章一个分类
tags 文章标签 一篇文章可以多个标签
  1. 如果 img 属性不填写的话，文章特色图会根据文章标题的 hashcode 的值取余，然后选取主题中对应的特色图片，从而达到让所有文章都的特色图各有特色。
  2. date 的值尽量保证每篇文章是唯一的，因为本主题中 Gitalk 和 Gitment 识别 id 是通过 date 的值来作为唯一标识的。
  3. 如果要对文章设置阅读验证密码的功能，不仅要在 Front-matter 中设置采用了 SHA256 加密的 password 的值，还需要在主题的 _config.yml 中激活了配置。有些在线的 SHA256 加密的地址，可供你使用：开源中国在线工具、chahuo、站长工具。

2023-01-19：笔记标题要正确，不要含有一些特殊字符，比如引号!!!!

05 代码

2023-01-17 为什么html在网页中显示为markup？

原因：HTML指的是超文本标记语言 (Hyper Text Markup Language)。HTML不是一种编程语言，而是一种标记语言(markup language)。“超文本”就是指页面内可以包含图片、链接，甚至音乐、程序等非文字元素。标记语言(或超文本标签语言)的结构包括“头”部分、和“主体”部分，其中“头”部提供关于网页的信息，“主体”部分提供网页的具体内容。

2023-01-17 yml和yaml

yml是yaml(YAML Ain’t Markup Language)语言的文件，以数据为中心，比xml等更适合做配置文件。

2023-01-19补：

上面的方法并未完全解决笔记图片问题，图片的路径是绝对路径，网站上无法显示，Typora能正常显示。后改进，在_post文件夹下新建medias，再往里新建post_imgs，与主题的medias一样，这样相当于Typora里调的是_post这边的图片，二网站调的是主题source内部的图片，只是要麻烦及时将两边的图片手动同步，但问题算完全解决。然后typora里面图片的绝对路径改为相对路径，类似这样：./medias/post_imgs/image-20230121180645258.png

2023-01-24：为了保证图片的命名整齐，最好typora只够复制本地图片时另存，不要把网络图片复制另存也勾起来，因为网络上的图片它的路径是很乱的，爬下来在本地再去复制。

13 优化文章路径

参考：

  • MateMaster的url优化 https://matemaster.cn/posts/52745.html

  • https://blog.csdn.net/qq_39195042/article/details/85112114

安装插件 npm install hexo-abbrlink --save

根目录_config.yml配置，添加：

abbrlink:
  alg: crc16  #算法选项：crc16、crc32，区别见之前的文章，这里默认为crc16丨crc32比crc16复杂一点，长一点
  rep: dec    #输出进制：十进制和十六进制，默认为10进制。丨dec为十进制，hex为十六进制

修改：

#permalink: :year/:month/:day/:title/
permalink: posts/:abbrlink.html

这个配置完成之后，文章的链接都变成了undefined,新的文章没问题，老的文章就不行了。这个问题其实我们仔细想一下就能明白，我们首先要执行hexo clean 清楚掉以前生成的文章缓存，然后hexo g重新渲染就ok了。

hexo s浏览博客时，提示4000端口问题

FATAL Port 4000 has been used. Try other port instead.

解决方案：

法一：可以在站点下的文件_config.yml内加上如下代码更改hexo-server运行时的端口号：

server:
  port: 4001 #端口号随意，默认是4000
  compress: true
  header: true

法二：在启动服务加上端口号：

hexo server -p 4001

butteryfly主题报错

extends includes/layout.pug block content include ./includes/mixins/post-ui.pug #recent-posts.recent-posts +postUI include includes/pagination.pug

原因：如果你沒有 pug 以及 stylus 的渲染器会报错

解决方案：

npm install hexo-renderer-pug hexo-renderer-stylus --save

hexo clean && hexo g && hexo s
